# Phase 1 Output Template

Codex may use this as the minimum structure for each Phase 1 document.

## Executive finding

State the answer first, including the recommended decision and why it is preferred.

## Evidence reviewed

List repository files, current code paths, official documentation, licences, benchmarks and deployment evidence.

## Current behaviour

Describe what exists today without assuming intended behaviour.

## Options considered

Compare credible alternatives using consistent criteria.

## Recommendation

Identify:

- selected option;
- rejected options and reasons;
- scope limitations;
- dependencies;
- migration impact;
- validation required.

## Risks and controls

Assign severity, likelihood, mitigation, owner and release gate.

## Decisions required from Aaron

List only decisions that materially affect architecture, engineering scope, commercial use or delivery.

## Acceptance criteria

Use testable statements. Avoid vague terms such as robust, accurate or production-ready without measurable evidence.

## References

Prefer official documentation, primary repositories, standards metadata and reproducible benchmark sources. Record access date and versions for changeable dependencies.
