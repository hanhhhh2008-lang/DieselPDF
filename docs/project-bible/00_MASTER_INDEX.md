# DieselPDF Structural Autodrafting — Project Bible

## Purpose

This folder is the durable project context for Codex, GPT and future engineering agents working on DieselPDF. It records the product goal, system boundaries, architecture, data model, coordinate and grid concepts, document adapters, FEA strategy, Australian Standards layer, QA framework, human inputs and staged development plan.

## Governing principle

The engineering dataset is the single source of truth. PDF, Canvas, CAD, IFC, FEA and AI are adapters, views or services around the dataset.

## Current status

The repository contains a working Windows-oriented Python/Tkinter prototype. The next authorised activity is Phase 1 architecture review only. No autonomous structural design or construction issue capability currently exists.
